# Dashboard 预览页图片上传设计

## 目标

让用户能在 Dashboard 的平台预览编辑流程中选择、拖入或粘贴一张本地图片，并将 Markdown 图片引用插入当前光标或选区。

## 范围

- 支持 JPG、PNG、WebP，单文件最大 10MB。
- 支持文件选择、拖入和剪贴板粘贴；一次处理一张图片。
- 图片复制到对应文章目录的 `images/`，Markdown 使用相对路径 `images/<文件名>`。
- 支持替换和删除已插入图片；删除仅清理没有任何 Markdown 引用的本地文件。
- 上传失败时保留编辑内容并显示明确错误。

## 非目标

- 不提供远程图床、跨文章素材库、批量上传、裁剪、压缩或替代文本编辑。
- 不支持 GIF、SVG 或超过大小限制的文件。

## 架构

Dashboard 客户端在预览编辑界面提供上传按钮、拖放目标和粘贴监听。它将 `videoId`、平台和图片以 `multipart/form-data` 提交给 Dashboard 服务端。服务端验证路径、MIME 类型和大小后，将文件安全地写入文章目录，并返回相对路径；客户端把该路径写成 Markdown 图片引用。

图片替换和删除复用同一套上传与 Markdown 更新逻辑。服务端删除前扫描目标 Markdown，只有旧文件没有其他引用时才删除，避免破坏被复用的图片。

## 错误与安全

- 只接受受支持格式和不超过 10MB 的单张图片。
- `videoId` 继续使用既有安全校验；文件名由服务端生成或净化，拒绝路径分隔符与遍历片段。
- 所有落盘路径必须验证在该视频的文章目录内。
- 上传错误以 JSON 返回，客户端显示 toast，不覆盖正文。

## 验证

测试将覆盖成功上传和相对路径返回、格式与大小拒绝、非法视频路径拒绝、重名文件避让，以及替换/删除不会误删仍被引用图片的规则。最后运行 Dashboard 定向测试与全量 TypeScript 类型检查。
